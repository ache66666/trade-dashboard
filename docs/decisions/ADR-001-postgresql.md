# ADR-001：运行时数据库迁移到 PostgreSQL

- 状态：Accepted
- 日期：2026-07-11

## 背景

项目最初使用本地 SQLite 文件保存指标和宏观事件。产品目标变为云端持续运行，页面不能依赖开发者电脑或某个本地磁盘文件。

## 问题

- Render 实例文件系统不适合作为长期、唯一的业务数据存储。
- SQLite 文件难以在多实例、备份、权限和远程维护场景中扩展。
- 本地数据库会让开发、部署和正式数据边界模糊。

## 方案

考虑过：继续携带 SQLite、使用平台持久磁盘、迁移到托管 PostgreSQL。选择 Supabase PostgreSQL，并保留 SQLite 作为只读备份和一次性迁移来源。

## 最终决定

应用运行时只通过 `DATABASE_URL` 访问 PostgreSQL。schema 由显式 SQL 管理，启动时不自动建表或种数据。原始数据一次性事务迁移并校验，SQLite 不再参与线上运行。

## 影响

### 正面

- 云端运行不依赖本地电脑。
- 数据可持久化、远程备份，并支持独立 Production/Staging。
- 参数化查询和数据库约束提供更稳定的数据边界。

### 代价与约束

- 必须安全管理连接串和连接池。
- 数据库不可用会直接影响应用，需要健康检查和运行监控。
- schema 变更必须执行正式迁移流程。
